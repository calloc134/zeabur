## GitHub の個人リポジトリの展開

### ステップ 1: サービスの作成

展開したいサービスを選択し、「サービスを作成」をクリックすると、`Git`と`Marketplace`の 2 つのオプションが表示されます。 この場合は、`Git`を選択します。

![サービスの作成](/deploy/intergration/add-service.png)

![サービスの選択](/deploy/intergration/select-service.png)

### ステップ 2: 個人リポジトリの紐づけ

この時点で、検索ボックスと`Configure GitHub`ボタンが表示されます。 ボタンをクリックし、ページの指示に従って個人リポジトリへのアクセスを承認してください。

![Githubとの紐づけ](/deploy/intergration/configure-github.png)

### ステップ 3: リポジトリの選択

承認後、検索ボックスが表示されます。 展開したいリポジトリの名前またはキーワードを入力して検索ボタンをクリックし、リポジトリを選択します。

![リポジトリの選択](/deploy/intergration/select-repo.png)

## GitHub 組織のリポジトリの展開

### ステップ 1: サービスの作成

前のセクションの[ステップ 1: サービスの作成](./github#step-1-create-a-service)と同じ手順です。

### ステップ 2: 組織リポジトリの紐づけ

この時点で、検索ボックスと`Configure GitHub`ボタンが表示されます。 ボタンをクリックし、表示されるページでプロンプトに従って、組織リポジトリへのアクセスを承認してください。 インストールプロセスでは、必要に応じてすべてまたは一部のリポジトリにアクセスを承認できます。

![Github組織との紐づけ](/deploy/intergration/configure-org.png)

### ステップ 3: リポジトリの選択

{/* prettier-ignore */}
<Callout>
  1. 組織の所有者ではなくメンバーである場合、組織を紐づけるのに十分なアクセス許可を持っていない可能性があります。しかし、アクセス許可を持っているすべてのリポジトリを読み取ることができ、GitHub 認証ページでリポジトリをすでに設定済みであることが表示されます。
  2.  組織のリポジトリの所有者またはメンバーではなく、外部コラボレーターである場合は、組織を紐づけるための十分なアクセス許可がない可能性があります。しかし、アクセス許可があるリポジトリを読み取ることができ、GitHub認証ページでリポジトリをすでに設定済みであることが示されます。
</Callout>

承認後、検索ボックスが表示されます。 展開したいリポジトリの名前またはキーワードを入力して検索ボタンをクリックし、リポジトリを選択します。

![組織のリポジトリの選択](/deploy/intergration/select-org-repo.png)

## 更新

### リポジトリの変更

個人/組織リポジトリの変更については、操作に応じて自動的にコードリポジトリを更新します。

### 組織メンバーの変更

組織メンバーの変更については、`GitHub Webhook`イベントをリアルタイムでリッスンし、リポジトリのアクセス権限が`GitHub`と一貫していることを確認します。

## アンインストール

### 個人リポジトリ

#### 完全なアンインストール

`Zeabur`を完全にアンインストールする必要がある場合は、GitHub の `Intergrations` - `Applications` - `Installed Github Apps`で`Zeabur`を見つけ、`Configure` をクリックし、`Danger zone`ページに表示される`Uninstall "Zeabur"`をクリックします。

![Full Uninstall](/deploy/intergration/uninstall-zeabur.png)

#### 認可の取り消し

後で`Zeabur`を再認証するために認可を取り消す必要がある場合は、GitHub の `Intergrations` - `Applications` - `Authorized Gtihub Apps`で`Zeabur`を見つけ、`Revoke`をクリックし、ポップアップページでリスクに同意してください。

![Revoke Authorization](/deploy/intergration/revoke-zeabur.png)

### 組織リポジトリ

[個人リポジトリ](./github#personal-repository)と同様ですが、加えて Github の`Settings` - `Switch to another account`で操作する組織を選択する必要があります。
